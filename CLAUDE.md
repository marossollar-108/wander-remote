## Project Overview

Wander Remote is a remote desktop control system. A **Host** app runs on the controlled computer, streams screen frames over WebSocket to a **Relay Server**, which bridges them to a **Viewer** web app in the browser. Input events flow in reverse.

## Architecture

```
Host (Electron/Python)  <--WebSocket-->  Relay Server  <--WebSocket-->  Viewer-UI (React PWA)
```

- **Host** captures screen, sends JPEG frames; receives and executes input events (mouse/keyboard)
- **Relay** authenticates, rate-limits, and routes messages between host and viewer
- **Viewer** renders frames on canvas, captures user input, sends normalized coordinates (0.0-1.0)

Two host implementations exist: **Electron host** (primary, actively maintained) and **Python host** (alternative, lighter weight).

## Build & Run Commands

### Electron Host
```bash
cd electron-host && npm install
npm start              # Development
npm run build          # All platforms
npm run build:mac      # macOS only
npm run build:win      # Windows only
```

### Relay Server
```bash
cd relay-server && npm install
npm start              # Runs on port 8765
# Or via Docker:
docker build -t wander-relay . && docker run -p 8765:8765 wander-relay
# Or via PM2:
pm2 start ecosystem.config.cjs
```

### Viewer-UI
```bash
cd viewer-ui && npm install
npm run dev            # Dev server on http://localhost:6100
npm run build          # Production build
```

### Python Host
```bash
cd host
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python host_gui.py
# Build standalone: python build.py
```

### Installer Scripts (for Python host)
```bash
cd installers
./build-mac.sh         # macOS
./build-linux.sh       # Linux
build-windows.bat      # Windows
```

## No Test Suite or Linter

There are no automated tests or linting configurations. Manual testing only via `viewer/viewer.py` and `host/test_input.py`.

## WebSocket Protocol

Messages are JSON with a `type` field. Binary frames are forwarded directly.

**Host→Relay:** `host_register`, `frame_full`, `frame_delta`, `ping`
**Viewer→Relay:** `viewer_connect`, `mouse_move`, `mouse_click`, `mouse_scroll`, `key_event`, `key_combo`, `quality_change`, `ping`
**Relay→Host:** `session_created`, `viewer_joined`, `peer_disconnected`, forwarded input events
**Relay→Viewer:** `connect_success`, `frame_full`, `frame_delta`, `peer_disconnected`, `error`, `pong`

Mouse coordinates are normalized (0.0-1.0). Validation functions live in `shared/protocol.py`. The relay JS port duplicates this logic in `relay-server/server.js`.

## Identity & Authentication

- **Host ID**: 6-digit number derived from MAC address (deterministic, persisted in `~/.wander-remote/host_id`)
- **Password**: 6-char hex, generated by relay on first registration, persisted in `relay-server/devices.json`
- Viewer auto-connect via URL params: `?session=XXXXXX&password=XXXXXX`

## Security

- Rate limiting: 100 connections/min per IP, 5 failed auth attempts/min
- Session timeout: 5 minutes inactivity
- Blocked key combos on host (Alt+F4, Ctrl+Alt+Del)
- Production uses WSS via Nginx reverse proxy (`relay-server/deploy/nginx-relay.conf`)

## Environment Variables

- **Relay:** `PORT`, `MAX_SESSIONS`, `SESSION_TIMEOUT_MS`, `RATE_LIMIT_PER_MIN` (see `.env.example`)
- **Viewer-UI:** `VITE_RELAY_URL` (default: `wss://remote.wanderhub.tech/ws`)
- **Python Host:** `RELAY_URL`, `CAPTURE_QUALITY`, `CAPTURE_MAX_FPS` (see `.env.example`)

## Platform-Specific Notes

- **macOS host input**: Electron uses robotjs; Python host uses Quartz/CoreGraphics (not pyautogui)
- **macOS permissions**: Host needs Accessibility + Screen Recording permissions
- **Electron host**: Has system tray integration, auto-updater (electron-updater), hardened runtime with entitlements

## Design System

Brand colors, typography (Montserrat/JetBrains Mono), and component styles are defined in `WANDER_DESIGN.md`. Tailwind theme in `viewer-ui/tailwind.config.js` mirrors these tokens.

## Deployment
